# Facilitator Guide

This guide provides instructions for facilitators running the GitHub Copilot Bootcamp.

## Overview

The bootcamp is a 4-week program designed to help developers master GitHub Copilot through hands-on exercises and real-world scenarios.

## Pre-Bootcamp Setup

### 1. Repository Setup
- Create a repository from this template for your organization
- Customize organization-specific standards in `docs/org-standards/`
- Update `prompts/org-style-prompts.md` with your team's conventions
- Configure branch protection rules

### 2. Participant Access
- Ensure all participants have GitHub Copilot licenses
- Grant repository access (read/write for labs)
- Create a GitHub Team for the bootcamp cohort

### 3. Communication Channels
- Set up Slack/Teams channel or use GitHub Discussions
- Schedule weekly sync meetings
- Prepare Q&A session schedule

## Week-by-Week Facilitation

### Week 1: Scaffold & Foundation
**Goals**: Familiarize with Copilot, set up projects

**Activities**:
- Kick-off session (1 hour)
- Live demo: Copilot features
- Lab: Complete week1-scaffold exercises
- Daily check-ins

**Facilitator Tips**:
- Emphasize prompt engineering basics
- Show auto-completion vs. chat differences
- Encourage experimentation

### Week 2: Features, Docs & Security
**Goals**: Build features with security awareness

**Activities**:
- Security-first development demo
- Feature implementation lab
- Documentation generation practice
- Mid-bootcamp retrospective

**Facilitator Tips**:
- Highlight security scanning integration
- Demonstrate documentation best practices
- Review common security pitfalls

### Week 3: DevOps & Testing
**Goals**: Integrate testing and CI/CD workflows

**Activities**:
- Test-driven development with Copilot
- CI/CD pipeline creation
- Code review exercises
- Guest speaker (optional): DevOps expert

**Facilitator Tips**:
- Show test generation capabilities
- Explain workflow automation
- Discuss code quality metrics

### Week 4: Legacy Code Refactoring
**Goals**: Apply skills to real-world legacy code

**Activities**:
- Refactoring strategies session
- Work on `legacy/messy_module.py`
- Final project presentations
- Graduation & next steps

**Facilitator Tips**:
- Share refactoring patterns
- Encourage incremental improvements
- Celebrate achievements

## Evaluation & Metrics

### Participant Progress Tracking
- Daily reflection submissions
- Lab completion rates
- PR quality and feedback
- Engagement in discussions

### Success Metrics
- Copilot adoption rate (see `scorecards/`)
- Code quality improvements
- Development velocity changes
- Participant satisfaction scores

## Common Challenges & Solutions

### Challenge: Participants not using Copilot effectively
**Solution**: Pair programming sessions, prompt kata exercises

### Challenge: Varying skill levels
**Solution**: Buddy system, optional advanced labs, office hours

### Challenge: Security concerns
**Solution**: Review security guidelines, demonstrate safe practices
